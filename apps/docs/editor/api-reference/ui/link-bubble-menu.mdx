Contextual menu that appears when clicking a link.

## LinkBubbleMenu.Default

Pre-configured link bubble menu with edit, open, and unlink actions.

```tsx
import { LinkBubbleMenu } from '@react-email/editor/ui';

<LinkBubbleMenu.Default />
```

<ResponseField name="placement" type="'top' | 'bottom'" default="'bottom'">
  Position relative to the link.
</ResponseField>

<ResponseField name="offset" type="number" default="8">
  Distance from the link in pixels.
</ResponseField>

<ResponseField name="onHide" type="() => void">
  Called when the bubble menu is hidden.
</ResponseField>

<ResponseField name="className" type="string">
  Additional CSS class on the wrapper element.
</ResponseField>

## Compound components

For custom link bubble menus, compose with these sub-components:

| Component | Description |
|-----------|-------------|
| `LinkBubbleMenu.Root` | Base container — sets up context and positioning |
| `LinkBubbleMenu.Toolbar` | Wrapper that hides when editing mode is active |
| `LinkBubbleMenu.Form` | Inline form for editing link URLs (shown when editing) |
| `LinkBubbleMenu.EditLink` | Button that enters editing mode |
| `LinkBubbleMenu.Unlink` | Button to remove the link |
| `LinkBubbleMenu.OpenLink` | Button to open the link in a new tab |

### Example

Here's how `LinkBubbleMenu.Default` is composed internally — use this as a
starting point for your own custom link bubble menu:

```tsx
import { LinkBubbleMenu } from '@react-email/editor/ui';

<LinkBubbleMenu.Root placement="top" offset={8}>
  <LinkBubbleMenu.Toolbar>
    <LinkBubbleMenu.EditLink />
    <LinkBubbleMenu.OpenLink />
    <LinkBubbleMenu.Unlink />
  </LinkBubbleMenu.Toolbar>
  <LinkBubbleMenu.Form />
</LinkBubbleMenu.Root>
```

The `Toolbar` automatically hides when the user clicks `EditLink`, and the `Form`
appears in its place. When the user submits or cancels, the toolbar reappears.

You can also pass props to `Form` for custom validation and callbacks:

```tsx
<LinkBubbleMenu.Form
  validateUrl={(value) => {
    try {
      new URL(value);
      return null; // valid
    } catch {
      return 'Please enter a valid URL';
    }
  }}
  onLinkApply={(href) => console.log('Link applied:', href)}
  onLinkRemove={() => console.log('Link removed')}
/>
```

### Form props

<ResponseField name="validateUrl" type="(value: string) => string | null">
  Custom URL validator. Return `null` if valid, or an error message string.
</ResponseField>

<ResponseField name="onLinkApply" type="(href: string) => void">
  Called after a link is successfully applied.
</ResponseField>

<ResponseField name="onLinkRemove" type="() => void">
  Called after a link is removed.
</ResponseField>

<ResponseField name="className" type="string">
  Additional CSS class on the form element.
</ResponseField>

## Context

Use `useLinkBubbleMenuContext()` inside custom children of `LinkBubbleMenu.Root`
to access the editor, link state, and editing state:

```tsx
import { useLinkBubbleMenuContext } from '@react-email/editor/ui';

function CustomToolbarItem() {
  const { editor, linkHref, isEditing, setIsEditing } = useLinkBubbleMenuContext();

  return (
    <span>Current link: {linkHref}</span>
  );
}
```

| Field | Type | Description |
|-------|------|-------------|
| `editor` | `Editor` | The TipTap editor instance |
| `linkHref` | `string` | The current link's `href` value |
| `isEditing` | `boolean` | Whether the menu is in editing mode |
| `setIsEditing` | `(value: boolean) => void` | Toggle editing mode |
